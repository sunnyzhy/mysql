# 清理日志

## 1. 临时生效（立即执行，无需重启）

在 MySQL 命令行中执行以下命令，可立即开始自动清理：

```sql
-- 设置 binlog 日志保留天数为 7 天，立即生效，MySQL 重启后失效
SET GLOBAL expire_logs_days = 7;
-- 或者使用新版本语法（MySQL 8.0+）
SET GLOBAL binlog_expire_logs_seconds = 604800; -- 604800秒 = 7天

-- 执行后，MySQL 会自动删除 7 天前的日志文件
FLUSH LOGS;
```

## 2. 永久生效（修改配置文件，需要重启）

编辑 MySQL 配置文件（`/etc/my.cnf` 或 `/etc/mysql/my.cnf`）：

```ini
[mysqld]
# 设置 binlog 自动保留的天数（旧参数，MySQL 5.7及以下常用）
expire_logs_days = 7
# 推荐新版本使用此参数（MySQL 8.0+），单位：秒
binlog_expire_logs_seconds = 604800

# 限制单个 binlog 文件的大小（可选，默认 1GB）
max_binlog_size = 500M
```
修改保存后，重启 MySQL 服务：
```bash
systemctl restart mysqld
```

## 3. 编写自定义清理脚本 (Cron Job)

**示例脚本（clean_mysql_logs.sh）**：

```bash
#!/bin/bash
# 清理慢查询日志：只保留最近 3 天的内容
find /var/log/mysql/ -name "slow-queries.log" -mtime +3 -exec truncate -s 0 {} \;

# 执行 MySQL 命令，删除 7 天前的 binlog
mysql -u root -pYourPassword -e "PURGE BINARY LOGS BEFORE DATE_SUB(NOW(), INTERVAL 7 DAY);"
```

将此脚本添加到 Crontab 中即可实现定时执行：

```bash
# crontab -e
# 每天凌晨两点执行清理脚本
0 2 * * * /root/clean_mysql_logs.sh
```

自动清理策略设置好后，可以随时用 `SHOW VARIABLES LIKE 'expire_logs_days';` 或 `SHOW BINARY LOGS;` 来检查配置是否生效以及磁盘空间是否释放。
